## 1. アプリ概要

milkは、北海道・中標津発の「言い出しっぺ駆動」イベントアプリ。milkのLINE公式アカウントのリッチメニューに設置し、LIFFアプリ（Webアプリ）として運用する。飲み会だけでなく体験イベント（アウトドア、ワークショップ等）にも対応し、地域のつながりを育むプラットフォームを目指す。

### 対象ユーザー

- 中標津エリアの20〜40代を中心とした住民
- 飲み会や体験イベントに気軽に参加したい人
- 自らイベントを「言い出したい」人

### 動作環境

- LINE内ブラウザ（LIFF）をメインとしたモバイルファースト設計
- 外部ブラウザ（Safari / Chrome）でも動作

### 2.3 グローバルナビゲーション

画面下部に固定のタブバーを配置する。

```
┌──────────────────────────────────────────────┐
│  🏠 ホーム  │  🔍 探す  │  ＋ 言い出す  │  👤 マイページ  │
└──────────────────────────────────────────────┘
```

- **ホーム**: イベント一覧（トップ）
- **探す**: イベント検索・フィルタ
- **言い出す**: イベント作成フォーム（中央に大きく配置）
- **マイページ**: ユーザーダッシュボード

## 4. ブランドガイドライン

### 4.1 ブランドコンセプト

milkは「中標津の自然のように、フレッシュで温かいつながりを生む」をコンセプトに、カジュアルで親しみやすいデザインを採用する。北海道・中標津の広大な自然と酪農の街のイメージを、グリーンを基調とした清潔感のあるビジュアルで表現する。

### 4.5 トーン＆ボイス

#### 基本方針

milkのテキストは「友達に話しかけるような、気さくで温かい」トーンを基本とする。

| カテゴリ | 方針 | 例 |
|---------|------|-----|
| ボタンラベル | 動詞で始める。カジュアルに。 | 「参加する」「言い出す！」「シェアする」 |
| 見出し | 短く、わかりやすく | 「今後の予定」「つながった人」 |
| 空状態 | 前向きなトーンで次のアクションを促す | 「まだイベントがありません。最初の言い出しっぺになろう！」 |
| エラー | 丁寧に、具体的に | 「読み込みに失敗しました。もう一度お試しください。」 |
| 成功 | お祝いの気持ちを込めて | 「参加確定！当日を楽しみに待ちましょう」 |
| 確認 | 安心感を持たせる | 「申し込み内容を確認してください」 |

#### 避けるべき表現

- 堅すぎる敬語（「ご登録いただきありがとうございます」→「登録できました！」）
- 専門用語（「認証」→「ログイン」、「トランザクション」→ 使わない）
- ネガティブな表現のみのエラー（「失敗しました」→「失敗しました。もう一度お試しください。」）

#### milk固有の言葉遣い

| 一般的な表現 | milkでの表現 |
|-------------|-------------|
| イベントを作成する | 言い出す |
| 主催者 | 言い出しっぺ |
| 参加登録 | 参加する |
| イベント開催条件を満たす | 開催確定 |

|------|------------|---------|
| アバター | ユーザープロフィール画像（LINE/Google/アップロード） | 64x64px 円形 |
| 表示名 | ユーザー設定の表示名 | h2サイズ Bold |
| 参加回数 | 参加済みイベントの総数 | 「参加 N回」 |
| 言い出し回数 | 作成したイベントの総数 | 「言い出し N回」 |

### 5.3 今後の予定セクション

**表示条件**: 参加予定のイベント（開催日が未来、かつキャンセルしていないもの）

**並び順**: 開催日の昇順（直近が上）

**イベントカードの構成**:

```
┌──────────────────────────────────────┐
│ [🍻飲み会]              [開催確定]   │
│                                      │
│ 金曜の夜、語ろう会                   │
│ 📅 3/21(金) 19:00〜                 │
│ 📍 居酒屋○○                        │
│ 👥 6/8人                            │
└──────────────────────────────────────┘
```

**表示上限**: デフォルト3件。それ以上は「すべて見る」リンクで展開。

### 5.4 過去の参加セクション

**表示条件**: 参加したイベント（開催日が過去のもの）

**並び順**: 開催日の降順（最新が上）

**イベントカードの構成**（コンパクト版）:

```
┌──────────────────────────────────────┐
│ [🍻] 春の歓迎会         3/7(金)     │
│      👥 8人参加                      │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│ [🏕] スノーシュー体験    2/15(土)    │
│      👥 12人参加                     │
└──────────────────────────────────────┘
```

**表示上限**: デフォルト4件。「もっと見る」で全件表示ページへ遷移。

### 5.5 つながりセクション

**目的**: 同じイベントに参加したことのある人を「つながり」として可視化する

**表示条件**: 同じイベントに1回以上同席したユーザー

**並び順**: 同席回数の降順

**つながりカードの構成**:

```
┌──────────────────────────────────────┐
│ (👤) 佐藤さん    3回一緒  最近:3/7  │
│ (👤) 鈴木さん    2回一緒  最近:2/15 │
│ (👤) 高橋さん    1回一緒  最近:3/7  │
│ (👤) 山田さん    1回一緒  最近:1/20 │
└──────────────────────────────────────┘
  → もっと見る（全12人）
```

**データ仕様**:

| 項目 | 説明 |
|------|------|
| アバター | ユーザーのプロフィール画像（40x40px 円形） |
| 表示名 | ユーザーの表示名 |
| 同席回数 | 同じイベントに参加した回数（「N回一緒」） |
| 最近の日付 | 最後に同席したイベントの開催日 |

**表示上限**: デフォルト6人。「もっと見る」で全員表示。

### 5.6 移動手段の記録セクション

**目的**: 過去のイベントで選択した移動手段を記録・表示し、安全な帰宅を促す

**表示内容**:

```
┌──────────────────────────────────────┐
│ 🚗 移動手段の記録                    │
│                                      │
│ 直近の選択:                          │
│ ・3/7  春の歓迎会 → 代行利用        │
│ ・2/15 スノーシュー → 自家用車      │
│                                      │
│ [🚕 代行・タクシー情報を見る]       │
└──────────────────────────────────────┘
```

**表示条件**: 過去に移動手段を登録したイベントが1件以上ある場合に表示。ない場合はセクション自体を非表示。

## 付録

### A. レスポンシブ対応方針

本アプリはモバイルファースト（LIFFアプリ）のため、以下の方針で設計する。

| ブレイクポイント | 対象 | 方針 |
|----------------|------|------|
| 〜375px | 小型スマホ | 最小対応。フォントサイズ・余白を微調整 |
| 376〜428px | 標準スマホ | 基準デザイン |
| 429〜768px | 大型スマホ・タブレット | カード幅を拡張、余白を広げる |
| 769px〜 | PC（管理画面） | 運営ダッシュボードのみ2カラムレイアウト対応 |

### B. アニメーション・インタラクション

| 対象 | アニメーション | 時間 |
|------|--------------|------|
| ページ遷移 | スライドイン（右から左） | 300ms ease-out |
| モーダル表示 | フェードイン + スライドアップ | 250ms ease-out |
| ボタンタップ | スケール縮小 (0.97) | 100ms |
| カードタップ | 背景色を `#f8f8f8` に | 150ms |
| トースト通知 | 上からスライドイン → 3秒後フェードアウト | 300ms / 300ms |
| スケルトンUI | パルスアニメーション（明滅） | 1.5s infinite |
| ステッパー進行 | 線のグリーン塗りつぶし | 400ms ease |

### C. スペーシングルール

| 名称 | サイズ | 用途 |
|------|--------|------|
| xs | 4px | アイコンとテキストの間 |
| sm | 8px | バッジ内パディング、要素間の最小間隔 |
| md | 12px | カード間マージン、リスト項目間 |
| lg | 16px | カード内パディング、セクション内間隔 |
| xl | 24px | セクション間マージン |
| xxl | 32px | ページ上下のパディング |
